2.1  Admin Login — Single Gmail Authentication  
The admin panel must be accessible via a single, pre -approved Google (Gmail) account only. No other 
Google account should be able to log in. This is not a public -facing registration system.  
 
2.1.1  How It Works — Technical Implementation  
• Use Google OAuth 2.0 (via Firebase Auth or NextAuth.js) to show a 'Sign in with Google' button  
• When any user attempts to sign in, capture their Gmail address from the Google token  
• Compare the returned email against a hardcoded or environment -variable -stored 
ALLOWED_ADMIN_EMAIL  
• If the email matches — grant access to the admin dashboard  
• If the email does NOT match — immediately reject login and show an error message: 'Access 
Denied. You are not authorized to access this panel.'  
• Never show a password field — only Google OAuth button  
 
ONE pre -approved Gmail only. Goog le OAuth2 via Spring Security — no other account can login.

2.1.2  Implementation Steps  
1. Step 1: Register the project in Google Cloud Console → Enable Google OAuth API  
2. Step 2: Get Client ID and Client Secret → store in .env file  
3. Step 3: Add authorized redirect URIs in Google Console (e.g., 
https://yourdomain.com/api/auth/callback/google)  
4. Step 4: In your auth callback, check: if (session.user.email !== process.env.ADMIN_EMAIL) 
{ redirect to /unauthorized }  
5. Step 5: Wrap all admin pages with a middleware/guard that checks session on every request  
 
📌 NOTE: Store the allowed email in a .env f ile as: ADMIN_EMAIL=youremail@gmail.com — Never 
hardcode it directly in source code. This way it can be changed without redeploying.

Confidential — Internal Use Only      Page    SECTION 10 — RESPONSIVE DESIGN & ADDITIONAL NOTES   
 
10.1  Responsive Design  
• Website must work seamlessly on Desktop, Tablet, and Mobile  
• Mobile navbar collapses into a hamburger menu  
• Dropdowns become full -screen ove rlays on mobile  
• Image grids stack to 1 or 2 columns on smaller screens  
• Map section must resize correctly on all screen sizes  
 
10.2  Performance Requirements  
• Images must be compressed (use WebP format where possible)  
• Lazy loading for below -the-fold images  
• Page load time target: under 3 seconds on desktop  
• SEO -friendly: proper meta tags, alt text on images, semantic HTML  
 
10.3  Out of Scope (v1.0)  
• E-commerce / cart / payment gateway — NOT included in v1.0  
• Customer account login — NOT required  
• Live chat widget  — optional, can be added later  
• Multi -language support — NOT in v1.0
